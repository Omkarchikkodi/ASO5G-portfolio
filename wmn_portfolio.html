Assignment 1

Configure a wireless network simulation using different propagation models and observe their impact on interference, mobility, and link aggregation.
+

Objective

Configure a wireless network simulation using different propagation models (Free Space, Two-Ray Ground, Log-Distance) and observe their impact on interference levels, mobility model behaviour, and link (interface) aggregation performance.

Procedure

  • Initialize Topology using MininetWiFi.
  • Deploy one static AP and two wireless stations (sta1, sta2).
  • Configure multi-interface bonding on sta1 using Linux bonding driver in Round-Robin mode.
  • Apply linear mobility path to sta2 to simulate movement away from the AP.
  • Sequentially cycle through Free Space, Two-Ray Ground, and Log-Distance propagation models.
  • Capture link signal parameters and aggregated throughput using iw and iperf.

Python Simulation Script

from mininet.log import setLogLevel, info
from mn_wifi.net import MininetWiFi
from mn_wifi.cli import CLI
from mn_wifi.link import wlan
from mn_wifi.propagationModels import propagationModel
import time

def topology():
    net = MininetWiFi()
    ap1 = net.addAccessPoint('ap1', ssid='prop-test-ssid', mode='g', channel='1', position='50,50,0')
    sta1 = net.addStation('sta1', ip='10.0.0.1/24', position='45,45,0')
    sta2 = net.addStation('sta2', ip='10.0.0.2/24', position='10,50,0')
    net.configureWifiNodes()
    net.addLink(sta1, cls=wlan, intf='sta1-wlan1')
    sta1.cmd('ifconfig sta1-wlan1 10.0.0.11 netmask 255.255.255.0 up')
    net.build()
    ap1.start([])
    # Link Aggregation (Bond0)
    sta1.cmd('modprobe bonding mode=balance-rr miimon=100')
    sta1.cmd('ip link add bond0 type bond')
    sta1.cmd('ip link set sta1-wlan0 master bond0')
    sta1.cmd('ip link set sta1-wlan1 master bond0')
    sta1.cmd('ifconfig bond0 10.0.0.100 netmask 255.255.255.0 up')
    # Mobility
    sta2.addMobilityModel(time=0, model='Linear', min_x=10, max_x=120, min_y=50, max_y=50)
    # Propagation models
    models = ['FreeSpacePropagationModel', 'TwoRayGroundPropagationModel', 'LogDistancePropagationModel']
    for model_name in models:
        if model_name == 'FreeSpacePropagationModel':
            net.setPropagationModel(model=propagationModel, exp=2)
        elif model_name == 'TwoRayGroundPropagationModel':
            net.setPropagationModel(model=propagationModel, sL=1, systemLoss=1)
        elif model_name == 'LogDistancePropagationModel':
            net.setPropagationModel(model=propagationModel, exp=3.5, sL=1)
        time.sleep(3)
        print(sta1.cmd('iw dev sta1-wlan0 link'))
        print(sta1.cmd('iperf -c 10.0.0.2 -t 2 -i 1'))
    CLI(net)
    net.stop()

Assignment 2

Capture and analyze wireless traffic to understand beacon transmission, spectrum usage, and network telemetry metrics.
+

Objective

Capture and analyze wireless traffic to understand beacon transmission, spectrum usage, and network telemetry metrics including RSSI, noise floors, and channel frequencies.

Procedure

  • Start Traffic Monitor — attach a packet sniffer to the virtual wireless monitor interface (mon0).
  • Isolate Beacon Frames — filter for 802.11 management frames where subtype matches beacon transmissions.
  • Extract Network Telemetry — read signal parameters (RSSI, noise floor, channel frequency) from Radiotap headers.
  • Export Dataset — save sniffed frames into a .pcap file for subsequent analysis.

Python Simulation Script

from mininet.log import setLogLevel, info
from mn_wifi.net import MininetWiFi
from mn_wifi.cli import CLI
import time, os

def capture_experiment():
    net = MininetWiFi()
    ap1 = net.addAccessPoint('ap1', ssid='Telemetry-Test-AP', mode='g', channel='6', position='50,50,0')
    sta1 = net.addStation('sta1', ip='10.0.0.1/24', position='40,40,0')
    net.configureWifiNodes()
    net.build()
    ap1.start([])
    os.system('iw dev ap1-wlan0 interface add mon0 type monitor')
    os.system('ifconfig mon0 up')
    pcap_file = "wireless_telemetry.pcap"
    capture_cmd = "tshark -i mon0 -Y 'wlan.fc.type_subtype == 0x08' -c 50 -w {} &".format(pcap_file)
    os.system(capture_cmd)
    time.sleep(2)
    print(sta1.cmd('ping -c 5 10.0.0.1'))
    time.sleep(8)
    parse_cmd = "tshark -r {} -T fields -e wlan.ssid -e radiotap.dbm_antsignal -e wlan_radio.frequency | head -n 15".format(pcap_file)
    os.system(parse_cmd)
    CLI(net)
    net.stop()

Assignment 3

Configure a wireless client to perform Active and Passive scanning. Measure and compare discovery time, power consumption, and network response.
+

Objective

Configure a wireless client device to perform Active scanning and Passive scanning. Measure and compare discovery time, power consumption, and network response.

Procedure

  • Initialize topology with one AP and one client station.
  • Configure AP beacon interval at 100ms to anchor the passive discovery window.
  • Execute Active Scanning: broadcast Probe Request frames and time the discovery window.
  • Execute Passive Scanning: listen on each channel for Beacon frames without transmitting.
  • Record and compare discovery times, TX power state, and channel dwell time.

Python Simulation Script

from mininet.log import setLogLevel, info
from mn_wifi.net import MininetWiFi
from mn_wifi.cli import CLI
import time

def scanning_experiment():
    net = MininetWiFi()
    ap1 = net.addAccessPoint('ap1', ssid='Discovery-AP', mode='g', channel='1', position='50,50,0')
    sta1 = net.addStation('sta1', ip='10.0.0.1/24', position='40,40,0')
    net.configureWifiNodes()
    net.build()
    ap1.start([])
    time.sleep(2)
    # Active Scanning
    start_active = time.time()
    active_output = sta1.cmd('iw dev sta1-wlan0 scan')
    active_discovery_time = (time.time() - start_active) * 1000
    print("Active Scanning Discovery Time: {:.2f} ms".format(active_discovery_time))
    # Passive Scanning
    start_passive = time.time()
    passive_output = sta1.cmd('iw dev sta1-wlan0 scan passive')
    passive_discovery_time = (time.time() - start_passive) * 1000
    print("Passive Scanning Discovery Time: {:.2f} ms".format(passive_discovery_time))
    CLI(net)
    net.stop()

Assignment 4

Simulate a mobile node moving between two access points and observe the handover process.
+

Objective

Simulate or configure a mobile node moving between two access points or base stations and observe the handover process, including RSSI variation and AP reassociation.

Procedure

  • Instantiate Mininet_wifi with wmediumd for realistic link-layer emulation.
  • Deploy two APs with the same SSID over orthogonal channels (1 and 6) and a mobile station with background scanning profile.
  • Apply Log-Distance propagation with exponent 4.5 (dense urban environment).
  • Program a time-bound mobility trajectory from (0,30,0) to (100,30,0) over 22 seconds.
  • Execute handover monitoring loop — polling station link status every 3 seconds to observe RSSI changes and AP reassociation.

Python Simulation Script

import sys, time
from mininet.log import info, setLogLevel
from mn_wifi.cli import CLI
from mn_wifi.link import wmediumd
from mn_wifi.net import Mininet_wifi

def handover_monitor(sta, iface='sta1-wlan0', interval=4, duration=24):
    start_time = time.time()
    while time.time() - start_time < duration:
        info('*** Handover status at %.1fs\n' % (time.time() - start_time))
        info(sta.cmd('iw dev %s link' % iface))
        time.sleep(interval)

def topology(args):
    net = Mininet_wifi(link=wmediumd)
    sta1 = net.addStation('sta1', ip='10.0.0.1/8', position='20,30,0', bgscan="simple:1:-60:5")
    ap1 = net.addAccessPoint('ap1', ssid='handover-ssid', mode='g', channel='1', position='20,30,0')
    ap2 = net.addAccessPoint('ap2', ssid='handover-ssid', mode='g', channel='6', position='80,30,0')
    h1 = net.addHost('h1', ip='10.0.0.10/8')
    c1 = net.addController('c1')
    net.setPropagationModel(model='logDistance', exp=4.5)
    net.configureNodes()
    s1 = net.addSwitch('s1')
    net.addLink(ap1, s1); net.addLink(ap2, s1); net.addLink(h1, s1)
    net.startMobility(time=0)
    net.mobility(sta1, 'start', time=2, position='0,30,0')
    net.mobility(sta1, 'stop', time=22, position='100,30,0')
    net.stopMobility(time=24)
    net.build()
    c1.start(); s1.start([c1]); ap1.start([c1]); ap2.start([c1])
    time.sleep(3)
    handover_monitor(sta1, interval=3, duration=22)
    CLI(net)
    net.stop()

  • Outer Header: Routing IP addresses of cluster nodes (gNodeB → UPF).

Course Project
ASO5G — Adaptive Slice Orchestrator for 5G Networks

Problem Statement

In modern 5G networks, traffic patterns are highly dynamic due to the increasing number of connected devices and diverse applications. Static network slicing approaches fail to efficiently handle sudden congestion conditions, leading to packet loss, increased latency, degraded throughput, and poor Quality of Service. Most existing systems lack mechanisms for automatic standby slice activation and dynamic traffic redistribution during congestion scenarios. Therefore, there is a need for an intelligent orchestration framework capable of continuously monitoring slice conditions and adaptively managing traffic loads in real time.

Objectives

  • Design and implement a congestion-aware adaptive slice orchestrator for 5G networks.
  • Deploy multiple 5G network slices using Open5GS and Kubernetes.
  • Continuously monitor slice utilization, active sessions, active UEs, throughput, and latency.
  • Dynamically activate a standby slice during congestion conditions.
  • Perform intelligent traffic migration from overloaded slices to standby slices.
  • Visualize orchestration events and network metrics using a real-time web dashboard.
  • Improve resource utilization and maintain service continuity in 5G infrastructures.

Scope of the Activity

The scope of the ASO5G framework focuses on adaptive orchestration and congestion management in 5G core networks. The system is designed for simulated multi-slice 5G environments using Open5GS, Kubernetes, and UERANSIM. The project demonstrates:

  • Real-time monitoring of multiple network slices.
  • Congestion detection using utilization thresholds.
  • Dynamic standby slice activation.
  • Adaptive traffic redistribution mechanisms.
  • Real-time visualization through a Flask-based dashboard.
  • Kubernetes-based deployment and orchestration of 5G network functions.

The framework can be further extended for AI-driven orchestration, predictive congestion analysis, cloud-native scaling, and production-grade telecom infrastructures.

Methodology

The ASO5G (Adaptive Slice Orchestrator for 5G Networks) is designed to provide congestion-aware adaptive orchestration in a Kubernetes-based 5G core environment. The framework continuously monitors network slices and dynamically activates a standby slice whenever congestion is detected. Traffic migration is then performed to reduce overload and maintain Quality of Service (QoS). The methodology combines Open5GS, Kubernetes, UERANSIM, Python-based orchestration, and a Flask-based real-time monitoring dashboard.

Proposed Solution

The ASO5G framework introduces an adaptive orchestration mechanism capable of monitoring real-time slice utilization, detecting congestion based on utilization thresholds, activating standby slices dynamically, redistributing traffic from congested slices, and visualizing orchestration events through a web dashboard.

The framework supports the following network slices:

eMBB Slice
mMTC Slice
URLLC Slice
Standby Recovery Slice

Whenever a slice exceeds the congestion threshold, the orchestrator automatically scales the standby slice using Kubernetes deployments and redirects a percentage of sessions to maintain stability.

System Architecture

The ASO5G framework consists of the following major components:

Open5GS 5G Core Network
Kubernetes Cluster
UERANSIM UE & gNB Simulator
Python-Based Adaptive Orchestrator
Flask-Based Monitoring Dashboard
Traffic Load Simulator

The orchestrator communicates with Kubernetes services to monitor and manage slice states dynamically.
